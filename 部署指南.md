# 🏃 训练日记 - 部署指南

## 快速开始

### 方式一：本地部署（推荐个人使用）

#### macOS / Linux

1. **确保已安装 Node.js 18+**
   ```bash
   node -v  # 检查版本
   ```

2. **解压项目文件**
   ```bash
   cd training-journal
   ```

3. **运行启动脚本**
   ```bash
   chmod +x start.sh  # 首次运行需要添加执行权限
   ./start.sh
   ```

4. **访问网站**
   打开浏览器访问 http://localhost:3000

#### Windows

1. **确保已安装 Node.js 18+**
   下载地址: https://nodejs.org/

2. **解压项目文件**

3. **双击运行 `start.bat`**

4. **访问网站**
   打开浏览器访问 http://localhost:3000

---

### 方式二：云服务器部署（推荐分享给朋友）

#### 使用 Render（免费）

1. **注册 Render 账号**
   - 访问 https://render.com/
   - 使用 GitHub 账号登录

2. **创建 Web Service**
   - 点击 "New +" → "Web Service"
   - 连接你的 GitHub 仓库，或选择 "Upload your code"

3. **配置服务**
   - **Name**: training-journal
   - **Runtime**: Node
   - **Build Command**: `pnpm install && pnpm run build`
   - **Start Command**: `node server.js`
   - **Plan**: Free

4. **添加环境变量**
   - 点击 "Environment" 标签
   - 添加变量: `JWT_SECRET` = 你的随机密钥（建议 32 位以上随机字符串）

5. **部署**
   - 点击 "Create Web Service"
   - 等待部署完成（约 2-3 分钟）
   - 获得公网访问地址，可以分享给朋友！

## 手动部署步骤

如果不想使用脚本，可以手动执行：

```bash
# 1. 安装依赖
pnpm install

# 2. 构建前端
pnpm run build

# 3. 启动服务器
node server.js
```

## 配置文件说明

### .env 环境变量

复制 `.env.example` 为 `.env`，可以配置以下选项：

```bash
# JWT 密钥（生产环境必须修改！用于用户认证加密）
JWT_SECRET=your-super-secret-key-change-this-in-production

# 服务器端口（默认 3000）
PORT=3000

# 数据库文件路径（可选，默认使用项目目录下的 data.db）
# DB_PATH=/path/to/your/data.db
```

⚠️ **重要**: 生产环境务必修改 `JWT_SECRET`，使用随机生成的长字符串！

## 数据备份

所有数据存储在 SQLite 数据库文件中：

- **默认位置**: 项目目录下的 `data.db`
- **备份方法**: 直接复制 `data.db` 文件即可
- **恢复方法**: 将备份的 `data.db` 复制回项目目录

## 常见问题

### 1. 端口被占用

如果 3000 端口被占用，可以修改 `.env` 文件：
```bash
PORT=3001
```

### 2. 数据库权限错误

确保运行脚本的用户对项目目录有读写权限：
```bash
chmod 755 training-journal
```

### 3. 安装依赖失败

如果遇到 better-sqlite3 编译错误，尝试：
```bash
# macOS
xcode-select --install

# Linux (Ubuntu/Debian)
sudo apt-get install build-essential python3

# 然后重新安装
rm -rf node_modules
pnpm install
```

### 4. 如何修改默认端口

编辑 `.env` 文件：
```bash
PORT=8080  # 修改为你想要的端口
```
